    ```typescript
    case 'monthly': {
      const targetMonth = date.getMonth() + 1
      date.setDate(1)

    ```

    Why this specific pattern (from planning context override, important finding #1):
    1. `date.setDate(1)` first — prevents overflow when calling `setMonth()`. Without this, `new Date(2024, 0, 31).setMonth(1)` auto-advances to March because Feb 31 is invalid.
    2. `date.setMonth(targetMonth)` advances the month safely (day is now 1, always valid).
    3. `new Date(date.getFullYear(), targetMonth + 1, 0).getDate()` returns the last day of the target month. `date.getFullYear()` is used AFTER setMonth to handle Desember → Januari year rollover correctly (setMonth(12) rolls to January next year automatically).
    4. `date.setDate(Math.min(d, lastDay))` clamps original day `d` to the target month's last day.
    5. `d` from line 29 (`const [y, m, d] = current.split('-').map(Number)`) is the ORIGINAL day, untouched by mutations.
    6. `const date` stays `const` — all operations are mutations, no reassignment. Do NOT change it to `let`.
