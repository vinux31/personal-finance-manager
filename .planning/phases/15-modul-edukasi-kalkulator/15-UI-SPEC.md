## Spacing Scale

Declared values (Tailwind default scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gap inside button, tooltip arrow offset |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing — chart axis label margin, breadcrumb separator |
| md | 16px (`p-4`, `gap-4`) | Default container padding (`px-4` modul + kalkulator), card body padding, slider+input row gap |
| lg | 24px (`p-6`, `gap-6`) | Section padding di kalkulator (input panel, output panel), modul section break (after H2) |
| xl | 32px (`p-8`, `gap-8`) | Layout gap antara header breadcrumb dan prose body, kalkulator chart wrapper margin |
| 2xl | 48px (`py-12`) | Major section breaks di modul (between theory → studi kasus → quick-check) |
| 3xl | 64px (`py-16`) | Page-level top/bottom spacing di modul container, footer prev/next margin-top |

**Exceptions:**
- Modul prose `max-w-[65ch]` (~520px reading column, ~65 char) — D-17 LOCKED, semantic content width, not a multiple of 4
- Kalkulator slider thumb touch target: 24px×24px (default radix slider `--slider-thumb-size`) — meets ≥24px minimum without override
- Tooltip arrow: 10px (`size-2.5` = 10px) — Radix default; non-multiple-of-4 acceptable for visual indicator only
- Chart `ResponsiveContainer` aspect ratio: `16/9` desktop, `4/3` mobile — semantic ratio, not spacing

## Typography

**Hard limits:** ≤4 distinct font sizes per typography context, exactly 2 weights total (400 regular + 600 semibold). Italic-400 acceptable as variant of 400.

### Modul Prose (Fraunces serif — `font-serif`)

**Sizes used:** 18 / 20 / 28 / 36 (4 distinct sizes).

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body modul | 18px | 400 | 1.7 | `text-lg leading-relaxed font-serif` |
| Blockquote / pull-quote (variant of body) | 18px | 400 italic | 1.7 | `text-lg italic font-serif border-l-4 border-muted pl-6 my-8` |
| H3 sub-section | 20px | 600 | 1.3 | `text-xl font-semibold leading-snug font-serif mt-8 mb-3` |
| H2 section | 28px | 600 | 1.25 | `text-3xl font-semibold leading-snug font-serif mt-12 mb-4` |
| H1 modul title | 36px | 600 | 1.2 | `text-4xl font-semibold leading-tight font-serif` |
| Code inline (override mono) | 18px (inherits body) | 400 | 1.5 | `font-mono text-[0.85em] bg-muted px-1.5 py-0.5 rounded` (relative size, no new declared size) |

**Note on consolidation:** Blockquote uses the same 18px body size with italic variant — not a new size declaration. Inline code uses `text-[0.85em]` relative sizing inherited from parent body, so it does not introduce a 5th size.

### Chrome / Kalkulator UI / Breadcrumb (Geist Variable sans — `font-sans` default)

**Sizes used:** 14 / 18 / 28 / 36 (4 distinct sizes).

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Breadcrumb | 14px | 400 | 1.5 | `text-sm font-normal text-muted-foreground` |
| Kalkulator label (slider) | 14px | 400 | 1.5 | `text-sm font-normal tracking-wide uppercase text-muted-foreground` |

| Footer prev/next nav label | 14px | 600 | 1.4 | `text-sm font-semibold` |
| Tooltip definition body | 14px | 400 | 1.5 | `text-sm leading-relaxed` |
| Table cell (5-tahunan tabel) | 14px | 400 | 1.4 | `text-sm tabular-nums` |
| Table header | 14px | 600 | 1.4 | `text-sm font-semibold uppercase tracking-wide` |
| Kalkulator H2 panel | 18px | 600 | 1.3 | `text-lg font-semibold` |

**Weight policy:** Project uses exactly 2 weights — **400 regular** + **600 semibold**. Italic 400 acceptable for blockquote variant (single italic style of the same 400 weight). NO 300, 500, 700, 800 weights anywhere in modul/kalkulator UI.

**Visual distinction strategy without weight 500:**
- Slider labels: weight 400 + `tracking-wide uppercase` + `text-muted-foreground` for label/value separation
- Number input value: weight 400 + `tabular-nums` (monospaced digits) for visual rhythm and alignment
- Table headers: weight 600 + `uppercase tracking-wide` for header/cell separation
- Footer nav prev/next labels: weight 600 (emphasis as interactive affordance, consistent with section headings)

### Slider + number input combo row (D-06)

```
Label "SALDO AWAL"                          Rp 10.000.000  ← number input
↑ uppercase tracking-wide muted             ↑ tabular-nums right-aligned
Helper "Modal yang sudah kamu punya..."     w-32 (8rem)
[━━━━━━━━━●─────────────────────────]       bg-muted text-sm tabular-nums
↑ Slider full-width (flex-1)
```

- Label visual treatment: `text-sm font-normal tracking-wide uppercase text-muted-foreground` (no weight 500; distinction via tracking + uppercase + muted color).
- Number input visual treatment: `text-sm font-normal tabular-nums` (no weight 500; distinction via mono digits + right-alignment).
- Desktop layout: label + helper on row 1 left, number input on row 1 right (flex justify-between). Slider on row 2 full-width.
- Mobile layout: label + helper stack row 1, number input row 2 right-aligned, slider row 3 full-width. (D-06 hint: slider dominant + number input collapse — Claude's Discretion.)
- Number input: `<input type="number" inputMode="numeric" pattern="[0-9.]*">` with `id-ID` thousand-separator formatting on blur, raw number on focus.
- Min/max bounds: number input clamps to slider min/max identically (D-06).

### GlossaryTooltip component (D-12, D-13, D-14, D-15)

```jsx
<GlossaryTooltip term="dca">DCA</GlossaryTooltip>
```

Renders:

```jsx
<TooltipProvider delayDuration={150}>
  <Tooltip>
    <TooltipTrigger asChild>
      <span
        tabIndex={0}
        role="button"
        className="border-b border-dotted border-muted-foreground cursor-help underline-offset-4 focus:outline-none focus:ring-2 focus:ring-brand focus:ring-offset-2 rounded-sm"
      >
        {children}
      </span>
    </TooltipTrigger>
    <TooltipContent
      side="top"
      align="center"
      sideOffset={4}
      className="max-w-xs px-3 py-2 text-sm font-sans"
    >
      <div className="font-semibold mb-0.5">{label}</div>
      <div className="leading-relaxed">{definition}</div>
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

- **Trigger visual:** dotted underline `border-b border-dotted` (NOT solid), color `border-muted-foreground` (subtle gray), cursor `cursor-help` (?).
- **Touch/mobile (D-14):** Radix Tooltip default is hover-only. To enable tap-to-open on mobile, use `<Tooltip open={open} onOpenChange={setOpen}>` controlled state with `onClick` handler on trigger. Tap-outside to close handled by Radix `onPointerDownOutside`.
- **Keyboard:** `tabIndex={0} role="button"` with `aria-describedby` linking to tooltip content. Enter/Space opens tooltip. Esc closes.
- **Tooltip body font:** Geist Variable sans (`font-sans`), NOT Fraunces. Tooltip is chrome, not prose.
- **Tooltip body weights:** label uses `font-semibold` (600), definition uses `font-normal` (400). No weight 500 anywhere.
- **Max width:** `max-w-xs` (320px). Definitions are 1-2 sentences (D-13 contract).
- **Animation:** Inherit shadcn Tooltip default (fade-in + slide-in 150ms — `data-[state=delayed-open]:animate-in fade-in-0 zoom-in-95`).

## Empty / Loading / Edge States

| Scenario | Behavior |
|----------|----------|
| Modul page loaded, Fraunces font still fetching | Render prose with serif fallback (Georgia, 'Times New Roman', serif). `font-display: swap` allows immediate text visibility. NO blocking spinner. CLS minimized by sizing Fraunces fallback metrics close to Georgia. |
| Kalkulator initial render | Defaults applied: saldo awal Rp 10.000.000, setoran bulanan Rp 1.000.000, return 8%, tenor 10 tahun. Output computed synchronously. NO loading spinner. |
| Kalkulator slider drag (D-07) | Real-time recalc on every `onValueChange` event. No debounce. Math is O(n) where n = tenor × 12 ≤ 480 iterations — sub-millisecond. |
| Glossary term not found (typo in code) | Throw at build time via TypeScript exhaustive `Record<GlossaryTerm, …>` (D-13). At runtime impossible (type-safe). |
| Tooltip render at viewport edge | Radix Tooltip `collisionPadding={8}` auto-flips side (top→bottom, etc.). |
| Modul prev/next at boundary | Wrap-around (D-05): modul 6 next = modul 1, modul 1 prev = modul 6. Always render both buttons. |
| Tabel breakdown row count (D-09) | Filter `[5, 10, 15, 20, 25, 30, 35, 40]` to entries `≤ tenor`. Min 1 row (when tenor ≥ 5; if tenor < 5, table is empty — render placeholder text "Atur tenor minimal 5 tahun untuk lihat breakdown" `text-sm text-muted-foreground italic`). |

## Responsive Breakpoints

Tailwind defaults: `sm: 640px`, `md: 768px`, `lg: 1024px`, `xl: 1280px`.

### Modul page

| Breakpoint | Behavior |
|-----------|----------|
| `<sm` (mobile) | `max-w-[65ch]` is wider than viewport, so effective width = 100% with `px-4` (16px gutters). Breadcrumb wraps if needed. H1 stays 36px (acceptable mobile reading). |
| `≥md` (desktop) | Centered `max-w-[65ch] mx-auto`, generous whitespace. |

### Kalkulator page

| Breakpoint | Behavior |
|-----------|----------|
| `<md` (mobile + small tablet) | Single column stack: input card → output card → chart card → tabel card. `gap-6` between cards. Slider rows: label/helper row 1, input row 2, slider row 3. Chart aspect `aspect-[4/3]`. |
| `≥md` (desktop) | Grid `grid-cols-2 gap-8` for input + output side-by-side. Chart + tabel stay full-width below. Slider rows: label/helper row 1 left, input row 1 right, slider row 2 full-width. Chart aspect `aspect-[16/9]`. |
| `≥lg` (large desktop) | Container caps at `max-w-5xl` (1024px). No further widening. |

### Tabel 5-tahunan

| Breakpoint | Behavior |
|-----------|----------|
| `<sm` | Standard shadcn `<Table>` with `overflow-x-auto` wrapper if needed. Numbers stay tabular-nums right-aligned. |
| `≥sm` | Full-width table inside card, no scroll needed (4 columns fit). |

### GlossaryTooltip

| Breakpoint | Behavior |
|-----------|----------|
| All | Tooltip max-width `max-w-xs` (320px). On `<sm` viewport, Radix `collisionPadding={8}` ensures it never extends beyond viewport edges; auto-flips side. |
