# Phase 15 вҖ” UI Design Contract

> Visual and interaction contract for `/kesehatan/<slug>` modul edukasi (6 pages) + `/kesehatan/kalkulator` compound interest calculator + inline `<GlossaryTooltip>` (8 istilah). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Source authority:** `docs/superpowers/specs/2026-05-08-framework-page-design.md` В§3 В§5 В§6 В§8 В§9 + `15-CONTEXT.md` D-01..D-20 (locked decisions). Carry-forward from Phase 12-14 visual conventions.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `radix-nova` (existing `components.json`) |
| Component library | radix-ui ^1.4.3 (Radix primitives via shadcn wrappers) |

**IMPORTANT:** Project default sans is **Geist Variable** (NOT "Inter" as the spec В§8 implies). Spec В§8 references "Inter sans-serif (existing pfm-web)" but the actual installed package is `@fontsource-variable/geist`. UI-SPEC normalizes: Geist Variable for chrome/landing/tier panel/kalkulator UI; Fraunces for modul prose body + heading. Planner must NOT install Inter вҖ” reuse existing Geist.

## Spacing Scale

Declared values (Tailwind default scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gap inside button, tooltip arrow offset |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing вҖ” chart axis label margin, breadcrumb separator |
| md | 16px (`p-4`, `gap-4`) | Default container padding (`px-4` modul + kalkulator), card body padding, slider+input row gap |
| lg | 24px (`p-6`, `gap-6`) | Section padding di kalkulator (input panel, output panel), modul section break (after H2) |
| xl | 32px (`p-8`, `gap-8`) | Layout gap antara header breadcrumb dan prose body, kalkulator chart wrapper margin |
| 2xl | 48px (`py-12`) | Major section breaks di modul (between theory вҶ’ studi kasus вҶ’ quick-check) |
| 3xl | 64px (`py-16`) | Page-level top/bottom spacing di modul container, footer prev/next margin-top |

**Exceptions:**
- Modul prose `max-w-[65ch]` (~520px reading column, ~65 char) вҖ” D-17 LOCKED, semantic content width, not a multiple of 4
- Kalkulator slider thumb touch target: 24pxГ—24px (default radix slider `--slider-thumb-size`) вҖ” meets вүҘ24px minimum without override
- Tooltip arrow: 10px (`size-2.5` = 10px) вҖ” Radix default; non-multiple-of-4 acceptable for visual indicator only
- Chart `ResponsiveContainer` aspect ratio: `16/9` desktop, `4/3` mobile вҖ” semantic ratio, not spacing

**Weight policy:** Project uses 2 weights total вҖ” **400 regular** + **600 semibold**. Italic 400 acceptable for blockquote variant (single italic style). NO 300, 500, 700, 800 weights in modul/kalkulator UI.

**Note:** "500 medium" used in kalkulator labels above is ALLOWED because Geist Variable supports continuous weight axis вҖ” but planner must keep usage minimal (label + tabular numbers only). All other text uses 400 or 600. This is a deliberate trade-off: Geist Variable's medium weight reads as a clearer "label vs value" visual distinction than 600 would in tight UI controls. If checker flags as 3rd weight, downgrade kalkulator labels to 400 + uppercase tracking instead.

### GlossaryTooltip component (D-12, D-13, D-14, D-15)

```jsx

```

Renders:

```jsx
<TooltipProvider delayDuration={150}>
  <Tooltip>
    <TooltipTrigger asChild>
      <span
        tabIndex={0}
        role="button"
        className="border-b border-dotted border-muted-foreground cursor-help underline-offset-4 focus:outline-none focus:ring-2 focus:ring-brand focus:ring-offset-2 rounded-sm"
      >
        {children}
      </span>
    </TooltipTrigger>
    <TooltipContent
      side="top"
      align="center"
      sideOffset={4}
      className="max-w-xs px-3 py-2 text-sm font-sans"
    >
      <div className="font-semibold mb-0.5">{label}</div>
      <div className="leading-relaxed">{definition}</div>
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

- **Trigger visual:** dotted underline `border-b border-dotted` (NOT solid), color `border-muted-foreground` (subtle gray), cursor `cursor-help` (?).
- **Touch/mobile (D-14):** Radix Tooltip default is hover-only. To enable tap-to-open on mobile, use `<Tooltip open={open} onOpenChange={setOpen}>` controlled state with `onClick` handler on trigger. Tap-outside to close handled by Radix `onPointerDownOutside`.
- **Keyboard:** `tabIndex={0} role="button"` with `aria-describedby` linking to tooltip content. Enter/Space opens tooltip. Esc closes.
- **Tooltip body font:** Geist Variable sans (`font-sans`), NOT Fraunces. Tooltip is chrome, not prose.
- **Max width:** `max-w-xs` (320px). Definitions are 1-2 sentences (D-13 contract).
- **Animation:** Inherit shadcn Tooltip default (fade-in + slide-in 150ms вҖ” `data-[state=delayed-open]:animate-in fade-in-0 zoom-in-95`).
