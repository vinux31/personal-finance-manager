## Typography

Exactly 2 font weights are used in this phase: **400 (regular)** and **600 (semibold)**. No intermediate weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label / sublabel | 10px (text-[10px]) | 600 (semibold) | 1.5 |
| Heading (panel title) | 14px (text-sm) | 600 (semibold) | 1.2 |
| Sisa Aman label | 14px (text-sm) | 600 (semibold) | 1.5 |

Notes:
- Bill name: `text-sm font-semibold` (14px, 600) — visual distinction from due-date sub-text is achieved by size (text-sm vs text-xs), not weight. Consistent with amount column and panel heading.
- Due-date sub-text ("jatuh tempo hari ini", "3 hari lagi", "terlambat 2 hari"): `text-xs text-muted-foreground` (12px, 400) — same as date sub-text in Transaksi Terakhir.
- Bill amount (right-aligned): `text-sm font-semibold tabular-nums` (14px, 600) — same as transaction amount column.
- Sisa Aman amount: `text-sm font-semibold tabular-nums` — merah (`text-red-500`) jika negatif, default foreground jika positif/nol.
- Panel header ("Tagihan Bulan Ini"): `text-sm font-semibold` — identical to existing Panel `h3` (DashboardTab line 261).

Source: DashboardTab.tsx Transaksi Terakhir pattern (lines 141–162), Phase 2 UI-SPEC typography.

## Layout Contracts

### Visual Hierarchy

Primary visual anchor: urgency dot + bill name (left of each row); summary anchor: Sisa Aman row (bottom of panel).

### Dashboard Row 3 — UpcomingBillsPanel (full-width)

```
[ Panel: "Tagihan Bulan Ini" ]               ← full-width, col-span full, same rounded-lg border bg-card p-5 as existing panels

  [ Bill Row: urgency dot | name + sub-text | amount ]
  [ Bill Row: urgency dot | name + sub-text | amount ]
  [ ... ]

  ─────────────────────────────────────────────────────  ← border-t mt-2 pt-2

  Sisa Aman Bulan Ini          Rp X.XXX.XXX             ← label left, amount right, tabular-nums
```

UpcomingBillsPanel is inserted in `DashboardTab.tsx` immediately after the closing `</div>` of the `grid grid-cols-1 md:grid-cols-2` div (row 2), within the outer `space-y-6` wrapper. No changes to row 1 or row 2 structure.

### Bill Row Anatomy

```
[ dot (h-2 w-2 rounded-full bg-{urgency}) ]
[ min-w-0 flex-1 ]
  [ text-sm font-semibold ] ← bill name (e.g. "PLN Listrik")
  [ text-xs text-muted-foreground ] ← "jatuh tempo hari ini" / "3 hari lagi" / "terlambat 2 hari"
[ text-sm font-semibold tabular-nums ] ← Rp X.XXX.XXX (right-aligned)
```

Row container: `flex items-center gap-3 py-2.5`
List container: `divide-y` (same as Transaksi Terakhir)

Source: CONTEXT.md D-03, D-05, `<specifics>`, DashboardTab.tsx lines 141–162.

### Sort Order

Bills sorted ascending by `next_due_date` — most urgent first. Overdue bills (already past) appear at the top.

### Max Height / Scroll Behavior

If bill list exceeds 6 items, cap the visible area at `max-h-64 overflow-y-auto` with a subtle `scrollbar-thin` style. This prevents the panel from pushing Dashboard content excessively far down on mobile. 6 items × ~40px row height = ~240px = fits within max-h-64 (256px).

Source: CONTEXT.md `<decisions>` (Claude's discretion — "Max height / scroll behavior jika tagihan sangat banyak").

### Sisa Aman Row

```
[ "Sisa Aman Bulan Ini" text-sm font-semibold ]    [ Rp X.XXX.XXX text-sm font-semibold tabular-nums text-{color} ]
```

Divider: `border-t mt-2 pt-2` separating bill list from Sisa Aman summary.
Color: `text-red-500` if value < 0, `text-foreground` if value ≥ 0.

Source: CONTEXT.md D-06, D-07, `<specifics>`.

## State Contracts

| State | Trigger | Visual |
|-------|---------|--------|
| Loading bills | `isLoading` from `useUpcomingBills()` | Inline: `<div className="flex h-32 items-center justify-center text-sm text-muted-foreground">Memuat…</div>` — consistent with Empty component height |
| Empty — no bills this month | Zero results from `useUpcomingBills()` | `<Empty text="Tidak ada tagihan bulan ini." />` — panel still renders (D-12) |
| Bills present | One or more bills | `<ul className="divide-y">` with bill rows, followed by Sisa Aman divider row |
| Bill overdue / today | `next_due_date <= today` | Urgency dot: `bg-red-500`; sub-text: `text-red-600` with "jatuh tempo hari ini" or "terlambat N hari" |
| Bill due ≤ 7 days | `next_due_date <= today + 7` AND `> today` | Urgency dot: `bg-yellow-500`; sub-text: `text-yellow-600` with "N hari lagi" |
| Bill due > 7 days | `next_due_date > today + 7` | Urgency dot: `bg-gray-400`; sub-text: `text-muted-foreground` with "N hari lagi" |
| Sisa Aman positive | `sisaAman >= 0` | Amount `text-foreground` |
| Sisa Aman negative | `sisaAman < 0` | Amount `text-red-500` |
| Error loading bills | `isError` from `useUpcomingBills()` | Inline: `<div className="flex h-32 items-center justify-center text-sm text-muted-foreground">Gagal memuat tagihan. Coba lagi.</div>` |

## Due-Date Sub-Text Logic

Computed from `diff = differenceInDays(next_due_date, today)` where diff is integer days:

| Condition | Sub-text copy |
|-----------|---------------|
| `diff < 0` (overdue) | "terlambat {Math.abs(diff)} hari" |
| `diff === 0` (today) | "jatuh tempo hari ini" |
| `diff === 1` | "besok" |
| `diff > 1` | "{diff} hari lagi" |

Use `date-fns` `differenceInCalendarDays(parseISO(next_due_date), new Date())` — consistent with existing date-fns usage in the codebase.

## Interaction Contracts

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Dashboard loads | Tab mount | `useUpcomingBills()` fires — fetches `recurring_templates` where `is_active=true`, `type='expense'`, `next_due_date <= endOfMonth` for current user |
| Bill row display | Data ready | Rows sorted ascending by `next_due_date` — most urgent at top |
| Bill list scrolls | More than 6 bills | `max-h-64 overflow-y-auto` — vertical scroll within panel, no page-level scroll disruption |
| Sisa Aman computed | `useAggregateByPeriod` + `useUpcomingBills` both resolved | `sisaAman = monthly.income - monthly.expense - totalBills` — reactive via `useMemo` |
| No interactions on rows | Read-only phase | No click handlers, no hover state change, no mark-as-paid button (Phase 4) |
