**Tujuan:** Verifikasi bahwa tombol "Refresh Harga" di tab Investasi mengirim payload dengan `date` field berformat WIB (UTC+7), bukan UTC.

**Langkah-langkah:**

1. Buka Chrome/browser, navigasi ke https://kantongpintar.vercel.app
2. Login dengan akun production
3. Buka **Chrome DevTools** (F12 atau Cmd+Opt+I)
4. Klik tab **Network** di DevTools
5. Di filter bar, ketik `fetch-prices` atau pilih filter "Fetch/XHR"
6. Navigasi ke tab **Investasi** (atau route `/investasi`) di app
7. Klik tombol **"Refresh Harga"**
8. Di panel Network, klik request `fetch-prices` yang muncul
9. Klik tab **Payload** (atau "Request Payload") di detail request
10. Lihat JSON body — cari field `date`

**Expected result:**
- Field `date` berformat `"YYYY-MM-DD"` (string)
- Nilai `date` = tanggal WIB hari ini (2026-05-15 jika sebelum tengah malam WIB, atau 2026-05-16 jika sudah lewat tengah malam WIB tapi UTC masih -1 hari)
- WIB = UTC+7, jadi:
  - Jam 17:00 UTC (00:00 WIB next day) → `date` harus = besok (WIB), bukan hari ini (UTC)
  - Jam 15:00 UTC (22:00 WIB same day) → `date` harus = hari ini (sama antara WIB dan UTC)

**Cara verifikasi cepat (tanpa tunggu midnight crossing):**
- Inspect nilai `date` di payload
- Bandingkan dengan `new Date().toLocaleDateString('sv-SE', { timeZone: 'Asia/Jakarta' })` di Console DevTools
- Keduanya harus sama

**Console DevTools check:**
```javascript
// Jalankan di Console tab DevTools untuk dapat expected WIB date
new Date().toLocaleDateString('sv-SE', { timeZone: 'Asia/Jakarta' })
// Output: "2026-05-15" (atau tanggal WIB hari ini)
```

**Catat hasil:**
- Payload date value: ___________
- Console WIB date: ___________
- Match? YES / NO
  </action>
  <acceptance_criteria>
    - Request `fetch-prices` terlihat di DevTools Network panel setelah klik "Refresh Harga"
    - Payload JSON body mengandung field `date` dengan format `"YYYY-MM-DD"`
    - Nilai `date` di payload = output dari `new Date().toLocaleDateString('sv-SE', { timeZone: 'Asia/Jakarta' })` di DevTools Console
    - Response dari edge function adalah 200 OK (bukan CORS error atau auth error)
  </acceptance_criteria>
