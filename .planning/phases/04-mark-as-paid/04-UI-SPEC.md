## Layout Contracts

### Updated Bill Row Anatomy

Phase 4 adds the "Lunas" button to the right of each row, after the amount:

```
[ dot (h-2 w-2 rounded-full bg-{urgency}) ]
[ min-w-0 flex-1 ]
  [ text-sm font-semibold ] ← bill name (e.g. "PLN Listrik")
  [ text-xs text-muted-foreground ] ← "jatuh tempo hari ini" / "3 hari lagi" / "terlambat 2 hari"
[ text-sm font-semibold tabular-nums ] ← Rp X.XXX.XXX (right-aligned)
[ Button variant="outline" size="sm" h-auto py-0.5 px-2 text-xs ] ← "Lunas"
```

Row container: `flex items-center gap-3 py-2.5` — unchanged. The "Lunas" button is a flex child appended after the amount span. It does NOT disrupt the existing layout because the amount is not `flex-1` — it is a fixed-width text node.

### AlertDialog Layout (confirmation dialog)

Triggered when user clicks "Lunas" on any row:

```
AlertDialog
  AlertDialogContent
    AlertDialogHeader
      AlertDialogTitle: "Tandai sebagai lunas?"
      AlertDialogDescription: "Tandai [nama tagihan] sebagai lunas? Transaksi pengeluaran akan dibuat secara otomatis."
    AlertDialogFooter
      AlertDialogCancel: "Batalkan"
      AlertDialogAction variant="destructive": "Ya, Lunas"
```

- Only one dialog is active at a time. Use a single `selectedBill` state that holds the bill being confirmed (or `null` when closed).
- Dialog is not persistent — it unmounts after close.

### Sisa Aman Row (updated formula — Phase 4 refinement)

Layout unchanged. Formula updated:

```
[ "Sisa Aman Bulan Ini" text-sm font-semibold ]    [ Rp X.XXX.XXX text-sm font-semibold tabular-nums text-{color} ]
```

Source data: `useUnpaidBills()` (or updated `useUpcomingBills()` with paid-filter) — only unpaid bills are summed. Paid bills no longer appear in the list or the sum.

Source: 04-CONTEXT.md D-03.

## State Contracts

All Phase 3 states are inherited unchanged. Phase 4 adds the following new states:

| State | Trigger | Visual |
|-------|---------|--------|
| Loading bills | `isLoading` from `useUpcomingBills()` / `useUnpaidBills()` | `<div className="flex h-32 items-center justify-center text-sm text-muted-foreground">Memuat…</div>` — unchanged |
| Empty — no unpaid bills | Zero results (all paid or no bills this month) | `<div className="flex h-32 items-center justify-center text-sm text-muted-foreground">Tidak ada tagihan bulan ini.</div>` — same copy, panel still renders |
| Bills present | One or more unpaid bills | Bill rows with "Lunas" button per row |
| Awaiting confirmation | User clicked "Lunas" on a row | `AlertDialog` opens with bill name in description. Other rows remain visible and inert behind dialog overlay. |
| Submitting (in-flight) | User clicked "Ya, Lunas" — RPC in progress | `AlertDialogAction` button shows disabled state or loading spinner. Cancel button also disabled. Duration: typically < 500ms on good connection. |
| Mark-as-paid success | RPC returns success | Dialog closes → bill row removed from list → toast appears |
| Mark-as-paid error | RPC returns error | Dialog stays open (do not close). Show inline error inside AlertDialogDescription: "Gagal memproses. Coba lagi." — Cancel button re-enabled. |
| All bills paid | Last bill marked paid | List empties → empty state renders — "Tidak ada tagihan bulan ini." |
| Sisa Aman after paid | After any successful mark-as-paid | Sisa Aman recalculates from refetched unpaid bills — positive adjustment expected |

## Interaction Contracts

All Phase 3 interactions are inherited. Phase 4 adds:

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Click "Lunas" button | User taps/clicks "Lunas" on a bill row | `AlertDialog` opens with that bill's name in the description. `selectedBill` state set to that bill. |
| Click "Batalkan" in dialog | User dismisses | `AlertDialog` closes. `selectedBill` reset to `null`. No data changes. |
| Click outside dialog | Backdrop click | Same as "Batalkan" — dialog closes, no changes. |
| Click "Ya, Lunas" in dialog | User confirms | RPC `mark_bill_paid(templateId, uid, today)` is called. Button transitions to disabled/loading. |
| RPC success | Promise resolves | 1. Dialog closes. 2. Invalidate queries: `upcoming-bills`, `transactions`, `aggregate`. 3. Toast: "[nama tagihan] ditandai lunas". 4. Bill row disappears on next render (query refetch). |
| RPC error | Promise rejects | Dialog remains open. `AlertDialogDescription` shows "Gagal memproses. Coba lagi." `[Ya, Lunas]` re-enabled. User can retry or cancel. |
| Dashboard loads | Tab mount | `useUnpaidBills()` (or updated hook) fetches unpaid bills — excludes templates with `bill_payments` record for current month. |
| Sisa Aman recalculates | Query refetch after mark-as-paid | `sisaAman = income - expense - sum(unpaid bills)` — paid bills no longer reduce the safe amount. |
| Empty state after all paid | All bills marked paid | Empty state renders inside panel — "Tidak ada tagihan bulan ini." Sisa Aman row still renders showing full Sisa Aman with no bill deductions. |

**`useProcessRecurring` is NOT modified.** The dedup guard relies entirely on `next_due_date` being advanced by the RPC — since `next_due_date` is past the current month after mark-as-paid, `useProcessRecurring` will not process it again. This is the architectural guard from 04-CONTEXT.md D-04.
