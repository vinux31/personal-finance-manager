
Fallbacks if `db push` is non-interactive-blocked or flags differ in this project's CLI version:

- Fallback B: Direct psql apply (only if Supabase CLI commands fail):
  ```bash

  ```
  This bypasses Supabase's migration tracking, so should only be used if CLI fails. Document fallback choice in summary.

If `db push` prompts for confirmation and the environment is non-interactive, pipe `yes` or use `--yes` flag if supported:
```bash
npx supabase db push --local --yes || yes | npx supabase db push --local
```
