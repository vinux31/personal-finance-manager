## UAT-2 (CONS-02)

**Scenario:** Refresh Harga writes WIB-correct date.

### Step 1 — Investasi tab, Refresh Harga clicked

Navigated to Investasi tab. Clicked "Refresh Harga".

Network result:
```
[POST] .../functions/v1/fetch-prices => [FAILED] net::ERR_FAILED
```

Console error:
```
Access to fetch at 'https://rqotdjrlswpizgpnznfn.supabase.co/functions/v1/fetch-prices'
from origin 'https://kantongpintar.vercel.app' has been blocked by CORS policy:
Response to preflight request doesn't pass access control check:
The 'Access-Control-Allow-Origin' header has a value 'https://kantongpintar.app'
that is not equal to the supplied origin.
```

**Root cause:** Pre-existing CORS misconfiguration in Edge Function `fetch-prices` — allows `kantongpintar.app` but app is deployed at `kantongpintar.vercel.app`. Unrelated to CONS-02 WIB fix. The request body (including `date` field) never reaches the server.

### Step 2 — Code verification (fallback)

Source verification of the WIB fix:
- `src/queries/investments.ts` line ~111: `const today = todayISO()` (changed from `new Date().toISOString().slice(0, 10)`)
- `todayISO()` imported from date utility — returns `new Date().toLocaleDateString('sv-SE', { timeZone: 'Asia/Jakarta' })` format (YYYY-MM-DD in WIB)
- ESLint rule in `eslint.config.js` blocks regression: `no-restricted-syntax` AST selector for `new Date().toISOString().slice()`
- `pnpm lint` exits 0 with this rule active (verified in 07-06-SUMMARY.md)

**NOTE:** Live date param not observable from browser because Edge Function CORS blocks the request before it reaches Supabase backend. Code change verified in source code + ESLint enforcement.

**UAT-2 Verdict:** PASS-WITH-NOTES (code fix verified in source; live network verification blocked by pre-existing CORS misconfiguration in Edge Function)
